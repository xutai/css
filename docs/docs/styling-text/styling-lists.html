list-specific styles

            list-style-type
                type of bullets to use for the list
                    square - unordered list
                    circle - unordered list
                    number - ordered list 
                    letter - ordered list
                    roman numeral - ordered list 
                    
            list-style-position
                bullets appear inside or outside the list items, before the start of each item

            list-style-image
                use a custom image for the bullet, rather than a simple square or circle.

            vs background

            this property is a bit limited in terms of controlling the position, size, etc. | much better

            ul, ul li  padding-left: 2rem(20px);
            the list items are still lined up with the order list items and the description list descriptions
            but the list items have some padding for the background images to sit inside
            If we didn't do this, the background images would overlap with the list item text, 
            which would look messy.

            background-size 1.6rem(16px)
            We ideally want the bullets to be the same size as the list items (or very slightly smaller or larger).
            16px fits very nicely with the 20px padding
            16px plus 4px of space between the bullet and the list item text 

            background-repeat 
            By default,  repeat, background images repeat until they fill up the available background space.

list-style shorthand

            list-style
                list-style-type
                    square
                list-style-image
                    url()
                list-style-position
                    inside

            list-style: squre url() inside
            the values can be listed in any order
            , and you can use one, two or all three
            (the default values used for the properties that are not included are disc, none, and outside).

            If both a type and an image are specified, 
            the type is used as a fallback if the image can't be loaded for some reason.

controlling list counting

            count differently on an ordered list 

            e.g. starting from a number other than 1, 
            or counting backwards, 
            or counting in steps of more than 1.

start

            The start attribute allows you to start the list counting from a number other than 1

reversed

            If there are more list items in a reversed list than the value of the start attribute,
            the count will continue to zero and then into negative values.

value

            Even if you are using a non-number list-style-type, 
            you still need to use the equivalent numerical values in the value attribute.
