formatting readable css

            Some developers put all of the rules onto a single line
            Other developers prefer to break everything onto a new line
                i prefer this way
                CSS doesn't mind which one you use. 
                We mdn developers personally find it is more readable to have 
                each property and value pair on a new line.

comment your css

            add a block of comments between logical sections, 
                to help locate different sections quickly when scanning through
                or even give you something to search for to jump right into that part of the CSS
                for example: "||"

            What you should comment are the things where you made a particular decision for a reason
                You may have used a CSS property in a specific way to get around older browser incompatibilities

            add the URL of the tutorial to the comments

create logical sections in your stylesheet

            !important 

            common styling | general styles
                have all of the common styling first in the stylesheet.
                body,
                h1, h2, h3, h4, h5
                ul and ol
                the talbe properies
                links
            utility classes
                for example a class that removes the default list style for lists 
                we're going to display as flex items or in some other way.
            sitewide
                the basic page layout, the header, navigation styling
            store pages
                specific things, broken down by the context, page or even component in which they are used

avoid overly-specific selectors

            very specific selectors
                duplicate chunks of your CSS to apply the same rules to another element
            simple class
                apply that anywhere

            There will be times when making something more specific makes sense,
            however this will generally be an exception rather than usual practice.
            === 
            very specific selectors (exception)
            simple class (usual practice)

break large stylesheets into multiple smaller ones

            very different styles for distinct parts of the site
                a stylesheet that includes all the global rules
                smaller ones that include the specific rules needed for those sections

other tools that can help

css methodologies

            prefer 
                adopting one of the approaches already designed by the community and tested across many projects
            rather than
                come up with your own rules for writing CSS

oocss

            concept of Object Oriented CSS (OOCSS)
                separate your CSS into reusable objects, 
                which can be used anywhere you need on your site

            create custom CSS for the different places this pattern is used


            reusing CSS (like overriding)


            The work that Nicole Sullivan did in describing this approach and promoting it means that 
            even people who are not strictly following an OOCSS approach today
            will generally be reusing CSS in this way
            it has entered our understanding as a good way to approach things in general

bem

            block element modifier
                a block is a standalone entity such as a button, menu, or logo
                An element is something like a list item or a title that is tied to the block it is in
                A modifier is a flag on a block or element that changes the styling or behavior

                extensive use of dashes and underscores in the CSS classes
                strict naming conventions of BEM

            widely used in larger web projects and many people write their CSS in this way
