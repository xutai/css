Positioning allows you to take elements out of the normal document layout flow, and make them behave differently

introducing positioning

allow us to override the basic document flow behavior described above
  sitting on top of one another
  always remaining in the same place inside the browser viewport

static positioning

put the element into its normal position in the document layout flow

relative positioning

very similar to static positioning
except that once the positioned element has taken its place in the normal layout flow, you can then modify its final position
  including making it overlap other elements on the page

introducing top, bottom, left, and right

absolute positioning

the gap where the positioned element should be in the document flow is no longer there,
An absolutely positioned element no longer exists in the normal document layout flow
it sits on its own layer separate from everything else
  we can create isolated UI features that don't interfere with the position of other elements on the page
    popup information boxes and control menus
    rollover panels
    UI features that can be dragged and dropped anywhere on the page containing element

positioning contexts

containing element: dependent on the position property of the ancestors of the positioned element

static
  initial containing block
    this block has the dimensions of the viewport, and is also the block that contains the <html> element
    Simply put, the absolutely positioned element will be contained outside of the <html> element, and be positioned relative to the initial viewport.

positioning context: which element the absolutely positioned element is positioned relative to
  setting positioning on one of the element's ancestors

introducing z-index

overlap. positioning text.

order in the source: later win over former
stacking order: z-index(alias of z-axis), positive values move them higher up the. default all auto === 0

fixed positioning

browser viewport
positioned heading no longer appears in the document flow, so the rest of the content moves up to the top;
add margin to some element to change it

position sticky

newer, hybrid between relative and fixed
it is relatively positioned until it is scrolled to a certain threshold point, after which it becomes fixed
  (e.g. 10px from the top of the viewport)
    navigation bar

position: sticky along with top: 0, supporting browsers will stick the headings to the top of the viewport as they reach that position.
Each subsequent header will then replace the previous one as it scrolls up to that position.
by testing, looks like it can only replace the same element type, eg. h2 cannot replace h3, so they will stack up one another
