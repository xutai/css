a simple float example

how the float works?
is taken out of the normal flow of the document and the element with the float set on it (the div element in this case) stuck to the left-hand side of its parent container (body, in this case)
Any content that comes below the floated element in the normal layout flow will now wrap around it, filling up the space to the right-hand side of it as far up as the top of the floated element. There, it will stop.
reverse: margin-right, top, left
we can add a margin to the float to push the text away
we can't add a margin to the text to move it away from the floating
a floated element is taken out of normal flow, and the boxes of the following items actually run behind the float

clearing floats

clearing floats: stop the following element from moving up
clear: left, right, both (the following element not the floated element)

clearing boxes wrapped around a float

fix this: a tall box and a short paragraph, we add a wrapper div to the float and the element with clearfix right following the float, but after the wrapper div the following paragraph is still affcted

the clearfix hack

hack: a clever or elegant technical accomplishment or a temporary solution

traditional, clearfix hack
inserting some generated content after the box which contains the float and wrapping content, and setting that to clear both

using overflow

alternative: overflow property of the wrapper to value rather than visible
block formatting context(BFC): this wrapper
in certain cases you might find unwanted scrollbars or clipped shadows due to unintended consequences of using overflow

display:flow-root

modern: using display: flow-root (wrapper)
create a BFC without using hacks - there will be no unintended consequences
